# Finance Refactor Review

## Найденные проблемы

- Заработок преподавателя считался как полная стоимость занятия, без процента преподавателя.
- Старые занятия могли пересчитываться от текущей цены абонемента и текущих настроек преподавателя.
- Страница «Финансы» смешивала продажи, оплаты и заработок в неочевидные показатели.
- Возвращенные занятия нужно было сохранять в истории, но исключать из финансовых итогов.

## Что изменено

- В `Teacher` добавлено поле `teacher_share_percent`, по умолчанию 50%.
- В `Visit` добавлены поля финансового снимка: `lesson_price`, `teacher_share_percent`, `teacher_earning`, `school_earning`.
- При списании занятия финансовый снимок рассчитывается и сохраняется сразу.
- При возврате занятия сохраненные суммы не обнуляются, но `is_cancelled = true` исключает занятие из итогов.
- `/finance/summary` возвращает продажи абонементов, полученные оплаты, стоимость проведенных занятий, выплаты преподавателям, доход школы и средние показатели.
- `/finance/teacher-earnings` возвращает подробную расшифровку по преподавателям и занятиям.
- Страница «Финансы» перестроена под новые показатели и показывает распределение стоимости занятий между преподавателями и школой.

## Миграции

Миграции выполняются при старте backend для локальной SQLite-базы:

- добавляется `teachers.teacher_share_percent`;
- добавляются финансовые поля в `visits`;
- старые посещения получают расчетный финансовый снимок.

## Восстановление старых данных

Для старых посещений финансовый снимок восстановлен расчетным способом:

```text
lesson_price = membership.price / membership.total_lessons
teacher_earning = lesson_price * teacher_share_percent / 100
school_earning = lesson_price - teacher_earning
```

Если раньше цена абонемента или процент преподавателя менялись, восстановленные значения могут отличаться от фактической исторической ставки. Новые посещения сохраняют точный снимок на момент списания.

## Оставшиеся технические долги

- Нет полноценной системы миграций Alembic; для локального MVP используются безопасные startup-миграции SQLite.
- Frontend-тесты пока не добавлены отдельным test runner, проверка UI покрыта TypeScript/Vite build.
- Для production-деплоя желательно заменить локальный bearer-token секрет на управляемый секрет окружения.
